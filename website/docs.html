环境要求

在开始之前，请确保你的服务器或本地环境满足以下要求：

  • Node.js >= 16.0.0
  • MySQL >= 8.0
  • npm 或 yarn 包管理器
  • Git（用于克隆项目）
💡 提示

推荐使用 Node.js 18 LTS 版本，可获得更好的性能和稳定性。

安装部署

1

克隆项目

从 GitHub 克隆项目到本地

# 克隆仓库 git clone https://github.com/lijiajun20130726-ux/Lboke.git cd Lboke
2

创建数据库

登录 MySQL 并创建数据库，然后导入初始化脚本

# 登录 MySQL mysql -u root -p # 执行以下命令创建数据库 CREATE DATABASE boke1 DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci; # 退出后导入初始化脚本 EXIT mysql -u root -p boke1 < database/init.sql
3

安装后端依赖

进入后端目录，安装 Node.js 依赖

cd houduan npm install
4

安装前端依赖

进入前端目录，安装前端依赖

cd ../frontend npm install
5

启动服务

分别启动后端和前端服务

# 终端1 - 启动后端 (端口 3000) cd houduan npm run dev # 终端2 - 启动前端 (端口 5173) cd frontend npm run dev
💡 提示

打开浏览器访问 http://localhost:5173 即可看到博客首页。

配置说明

后端配置文件位于 houduan/.env，首次使用需要创建并配置：

数据库配置

DB_HOST=localhost DB_PORT=3306 DB_USER=root DB_PASSWORD=你的数据库密码 DB_NAME=boke1

JWT 配置

JWT_SECRET=你的JWT密钥(建议使用随机字符串) JWT_EXPIRES_IN=7d # Token 过期时间，可选：1d, 7d, 30d

邮件配置

如果需要发送邮件功能（如订阅通知），请配置 SMTP：

EMAIL_HOST=smtp.你的邮箱服务商.com EMAIL_PORT=465 EMAIL_USER=你的邮箱地址 EMAIL_PASS=你的邮箱密码或授权码
⚠️ 安全提醒

请勿将 .env 文件上传到 GitHub！该文件已在 .gitignore 中忽略。

前端配置

前端环境变量位于 frontend/.env.development 和 frontend/.env.production：

VITE_API_BASE_URL=http://localhost:3000/api

部署到生产环境时，请将 URL 改为你的实际后端地址。

管理后台

访问管理后台：http://localhost:5173/admin

默认管理员账号：admin / password

⚠️ 重要

首次登录后请立即修改默认密码！

后台功能

  • 仪表盘 - 查看网站数据统计
  • 文章管理 - 创建、编辑、删除文章
  • 分类管理 - 管理文章分类
  • 标签管理 - 管理文章标签
  • 评论管理 - 审核游客评论
  • 订阅管理 - 查看订阅用户列表
  • 友链管理 - 审核友链申请
  • 公告管理 - 发布网站公告
  • 网站设置 - 配置网站基本信息

文章管理

在后台的文章管理页面，你可以：

  • 创建文章 - 使用 Markdown 编辑器编写文章
  • 编辑文章 - 修改已发布的文章
  • 删除文章 - 将文章移至回收站
  • 置顶文章 - 将重要文章置顶显示
  • 设置分类 - 为文章选择分类
  • 添加标签 - 为文章添加多个标签

文章状态

  • 草稿 - 仅作者可见，不会公开显示
  • 已发布 - 对所有访客可见
  • 已删除 - 已移至回收站

Markdown 使用

编辑器支持标准 Markdown 语法，以及一些扩展功能：

# 一级标题 ## 二级标题 **加粗文本** *斜体文本* - 列表项 1. 有序列表 [链接文字](URL) ![图片描述](图片URL) ```代码块```

分类与标签

分类

分类用于对文章进行粗略分类，建议设置 3-5 个主要分类，例如：

  • 技术
  • 生活
  • 笔记

标签

标签用于对文章进行细化，一个文章可以有多个标签，例如：

  • Vue、React、TypeScript
  • Node.js、Python、Go
  • 数据库、设计模式、算法
💡 最佳实践

分类数量不宜过多，标签可以相对灵活，便于文章归类和搜索。

评论管理

博客支持游客评论功能，访客无需登录即可发表评论。

评论规则

  • 访客需要填写昵称和邮箱（邮箱不会公开显示）
  • 支持回复功能，可以对评论进行嵌套回复
  • 管理员可以在后台审核评论，设置是否显示

评论审核

在后台的评论管理页面，你可以：

  • 查看所有评论和回复
  • 审核评论是否显示
  • 删除不当评论

订阅功能

访客可以订阅博客，当有新文章发布时会收到邮件通知。

订阅流程

  1. 访客在博客首页输入邮箱地址
  2. 系统发送验证邮件到该邮箱
  3. 访客点击验证链接确认订阅
  4. 订阅成功后，新文章发布时会收到通知邮件
💡 注意

订阅功能需要正确配置邮件 SMTP 服务才能正常工作。

友链管理

博客支持开放友链申请，其他博主可以申请与您交换链接。

申请条件

  • 网站内容健康，遵守法律法规
  • 网站可以正常访问
  • 原创内容为主

申请流程

  1. 在博客友链页面填写申请表单
  2. 管理员在后台审核申请
  3. 审核通过后，友链将在页面上显示

常见问题

Q: 如何修改网站名称和描述？

A: 登录管理后台 → 网站设置，在这里可以修改网站名称、描述、作者信息等。

Q: 如何关闭游客评论？

A: 目前评论功能是固定的，如需关闭可以修改代码或联系开发者。

Q: 如何备份数据库？

A: 使用 MySQL 自带的 mysqldump 命令：

mysqldump -u root -p boke1 > backup.sql

Q: 如何部署到服务器？

A: 生产环境部署建议：

  • 使用 PM2 管理后端进程
  • 前端执行 npm run build 构建静态文件
  • 使用 Nginx 反向代理到后端服务
  • 配置 HTTPS（推荐使用 Let's Encrypt 免费证书）
